Messing with¶

  • Input Stream (istream), from keyboard to program, cin , known as EXTRACTION
  • Output stream(ostream). from program to monitor, cout, known as INSERTION Above is grouped together as iostream preprocessor
  • Reading and writing data to file, using the fstream preprocessor. And also divided in ifstream for reading file, and ofstream for wrting a file. Main point here, wrting a file is same as where the program outputs to monitor, so we ouputting/wrting to file. And reading a file, is same as keyboard inputing to program, so we inputing/reading (from file) to program

  • Serialization Main Point here is to 'write' store (serialize) information, then 'read' or unpack (deserialize) that information. So can file, buffer, class, socket...that will be the host or object that you will need to use. In our examples we overload the INSERTION and EXTRACTION operator, to allow us in essence to do fout<<s1, instead of writing/reading individual fields.

  • Binary files Main point here, is mode for reading and writng that we need to change. Nice discussion about why faster.

  • Manipulators Main point here what they are, e.g \n, same as endl. We get others for deal with hex, oct Dec, fixed, scientific numbers etc.

class Student{
public:
    string name;
    string role;
    int age;
    
    friend ofstream& EXTRACTION_OPERATOR(ofstream &fout, Student &s1);
    friend ifstream& INSERTION_OPERATOR(ifstream &fin, Student &s1);
    friend ostream&  EXTRACTION_OPERATOR(ostream &cot, Student &s1);
    
}
