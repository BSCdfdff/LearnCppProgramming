Messing with¶

  • Auto, The essence here is that we do know when declaring the variable, so the compiler will judge what datatype should be used, when iy does its calculation.

  • decltype() Main essence here, take the type of previously defined data type.

  • final keyword Two points here, 1) If placed with class, say parent class, then child class cannot inherit it 2) if placed with parent function, then child class cannot override it.

  • Lamda expressions, In format capture_list->return type {body} Main Purpose of using this, is to define a function whout a name, Thats the gist. More points here

  • Smart pointers Main point here, they are template (generic) pointers so they must be followed by type Three types: unique_ptr,shared_ptr, weak_ptr

    1. unique_ptr: cannot reassign to another pointer, unless you move it
    2. shared_ptr: Can reassign, without moving it, but maintains, ref counter which can lead to deadlocks
    3. weak_ptr, same as no 2, but is much weaker implementation of no 2.

  • Inclass initializer Main point here, can initialize variables at class level

  • Delegation of constructors Main point here: Constructors that can call other constructors.

  • Ellipses: Where number of parameters are not fixed
